FCI BMad Kit · Multi-repo Workspace

Hướng dẫn cài đặt

Thiết lập workspace đa repo cho đội FCI — PO, BA, Dev, Tester mỗi người một máy, điều phối qua Git.

📋 /fci-po 📊 /fci-ba ⚙️ /fci-dev 🧪 /fci-tester

📖 Tổng quan

FCI BMad Kit chạy trên mô hình multi-repo workspace: mỗi thành viên cài trên máy riêng, dữ liệu (PRD, Epics, Stories, handoff) chia sẻ qua một repo Git chung tên docs. Agent tự git pull/push, không ai cần biết lệnh git.

Kiến trúc workspace sau khi cài

# Layout trên máy của mỗi người ~/workspace/ ├── _bmad/ ← BMad + 4 agent FCI (cài ở root) ├── docs/ ← HUB CHUNG — repo Git chia sẻ cho cả đội │ ├── workspace.yml registry danh sách repo dự án │ ├── handoff/ handoff giữa PO→BA→Dev→Tester │ ├── PRDs/ Epics/ Stories/ │ └── tasks/ test-plans/ ├── octavia/ ← app repo (code) · có .codegraph/ └── octavia-dashboard/ ← app repo (code) · có .codegraph/ ┌── 1 MCP "codegraph --multi-repo" ──┐ │ phục vụ tất cả app repo; agent │ │ truyền param repo=<name> để chọn │ └──────────────────────────────────────┘

ℹ️
Tài liệu này chỉ hướng dẫn multi-repo workspace. Nếu cần cài single-repo, xem README.md.

✅ Yêu cầu trước khi cài

🪟
Windows: Script init-workspace.sh là bash — cần chạy trong Git Bash (cài cùng Git for Windows). Không dùng CMD hay PowerShell. Mở Git Bash rồi làm theo các bước bên dưới bình thường.
🟢
Node.js
≥ 22 (bắt buộc)
Codegraph dùng better-sqlite3 prebuilt cho Node 22. Dưới Node 22 sẽ lỗi lúc build.
🤖
Claude Code
CLI mới nhất
Cài từ claude.ai/code. Cần login bằng Anthropic account.
🔑
Git + HTTPS
GitLab access token
Cần Personal Access Token (scope: read_repository / write_repository) hoặc dùng credential helper để không nhập mật khẩu mỗi lần.

Kiểm tra nhanh

# Kiểm tra Node version (phải ≥ 22) $ node --version v22.13.0 # Kiểm tra Claude Code $ claude --version claude 1.x.x # Kiểm tra HTTPS đến GitLab (dùng Personal Access Token) $ git ls-remote https://gitlab.com/fci/lbaas-docs.git → liệt kê refs không lỗi = OK

Cấu hình credential helper (tránh nhập token mỗi lần)

macOS
# Lưu vào keychain macOS (an toàn, permanent) $ git config --global credential.helper osxkeychain
Windows (Git Bash)
# Dùng Windows Credential Manager (built-in, permanent) $ git config --global credential.helper manager
Linux / macOS (không dùng keychain)
# Cache trong bộ nhớ 8 giờ $ git config --global credential.helper "cache --timeout=28800"
ℹ️
Tạo Personal Access Token tại: GitLab → Settings → Access Tokens. Scope cần: read_repository + write_repository. Nhập token thay cho password khi Git hỏi lần đầu.
⚠️
Nếu Node chưa đủ version, script có thể tự cài qua nvm (đặt AUTO_INSTALL_NODE=1 — mặc định là on). Nhưng để chắc ăn, nên cài Node 22 thủ công trước.

🆕 Kịch bản A — Dự án mới (chưa có workspace.yml)

Dùng khi bạn là người đầu tiên thiết lập workspace cho dự án. Bạn sẽ khai báo repo docs, đăng ký các app repo, rồi push workspace.yml để đồng đội dùng sau.

💡
Ai nên chạy kịch bản này? Thường là Dev (có đủ quyền truy cập các repo) hoặc BA. Sau khi push workspace.yml, các thành viên khác dùng Kịch bản B.
🪟
Windows: Mở Git Bash (không dùng CMD/PowerShell) rồi chạy các lệnh bên dưới.
1
Tạo thư mục workspace và vào đó
$ mkdir ~/work/LBaaS && cd ~/work/LBaaS
💡
Đặt tên thư mục theo tên dự án. Script sẽ dùng tên này làm WORKSPACE_NAME trong workspace.yml.
2
Chạy init-workspace.sh

Script sẽ hỏi role của bạn và dẫn qua từng bước.

Cách 1 — Không cần clone kit trước (recommended)
# Chạy thẳng từ internet, script hỏi role tương tác $ WORKSPACE_MODE=1 \ bash -c "$(curl -fsSL https://raw.githubusercontent.com/vncharles/fci-bmad-kit/main/install.sh)"
Cách 2 — Đã clone kit về máy
$ git clone https://github.com/vncharles/fci-bmad-kit.git /tmp/fci-kit $ TARGET_DIR=~/work/LBaaS /tmp/fci-kit/init-workspace.sh
3
Chọn role khi script hỏi
Bạn là vai trò nào? 1) PO 2) BA 3) Dev 4) Tester Chọn (1-4): 3 → Role: dev → clone docs + app repo + codegraph
ℹ️
Nếu không muốn script hỏi, đặt sẵn ROLE=dev (po | ba | dev | tester) trước lệnh chạy.
4
Nhập URL repo docs

Script hỏi URL của repo docs trên GitLab. Repo docs phải được tạo sẵn trên GitLab (có thể rỗng).

[1] Repo docs (hub chia sẻ) GitLab URL cho repo docs: https://gitlab.com/fci/lbaas-docs.git Branch repo docs [main]: ⏎ • git clone → docs (branch: main) ✓ docs sẵn sàng. Registry: ~/work/LBaaS/docs/workspace.yml
⚠️
Tên folder phải là docs (cố định). Script sẽ clone repo về thư mục docs/ trong workspace. Khi Git hỏi username/password, nhập GitLab username và Personal Access Token (không phải mật khẩu GitLab).
5
Khai báo các app repo

Script liệt kê các repo đã có trong workspace.yml (lần đầu thì rỗng) và hỏi bạn muốn khai báo thêm không.

[3] App repo — khai báo và clone (workspace.yml chưa có app repo nào) Khai báo thêm một app repo mới? [n] y Tên repo (vd octavia): octavia GitLab URL cho 'octavia': https://gitlab.com/fci/octavia.git Branch cho 'octavia' [main]: ⏎ ✓ Đã ghi 'octavia' vào registry. • git clone → octavia ▶ Codegraph build cho repo: ~/work/LBaaS/octavia • Embed (jina-code / structured)... ✓ 'octavia' sẵn sàng (branch: main) Khai báo thêm một app repo mới? [n] y Tên repo: octavia-dashboard GitLab URL: https://gitlab.com/fci/octavia-dashboard.git Branch [main]: ⏎ ✓ 'octavia-dashboard' clone + build xong Khai báo thêm một app repo mới? [n] ⏎
6
Script tự đăng ký MCP + cài BMad

Không cần làm gì — script tự chạy hết 2 bước này.

[3.5] Đăng ký MCP codegraph (multi-repo) • claude mcp add codegraph → /Users/.../v22.x.x/bin/codegraph mcp --multi-repo ✓ MCP 'codegraph' đã đăng ký [4] Cài BMad ở workspace root • npx bmad-method install (non-interactive) • Module: bmm + tea + fci ✓ 4 agent sẵn sàng: /fci-po /fci-ba /fci-dev /fci-tester
7
Push workspace.yml lên GitLab

Bước này quan trọng — sau khi push, đồng đội chỉ cần URL repo docs để tham gia.

[5] Đồng bộ registry lên repo docs Commit & push workspace.yml lên repo docs cho đội? [y] ⏎ → git add/commit/push trong docs/ ✓ workspace.yml đã được push
✅
Hoàn tất! Gửi URL repo docs (https://gitlab.com/fci/lbaas-docs.git) cho đồng đội. Họ dùng URL này để chạy Kịch bản B.

🔗 Kịch bản B — Dự án đã có workspace.yml

Dùng khi workspace đã được ai đó thiết lập (Kịch bản A) và push workspace.yml lên GitLab. Bạn chỉ cần URL repo docs là đủ — script sẽ đọc workspace.yml và tự clone các repo cần thiết.

🪟
Windows: Mở Git Bash (không dùng CMD/PowerShell) rồi chạy các lệnh bên dưới.

Bạn cần biết

URL của repo docs — hỏi người đã setup workspace (Dev/BA). Ví dụ: https://gitlab.com/fci/lbaas-docs.git

Tạo thư mục workspace và vào đó
$ mkdir ~/work/LBaaS && cd ~/work/LBaaS

Chạy init-workspace.sh

⚡ Nếu role của bạn là PO

PO chỉ clone repo docs, không cần clone app repo và không cài Codegraph. Script sẽ tự bỏ qua các bước đó khi bạn chọn role po.

📄 workspace.yml — Registry dự án

File docs/workspace.yml là "danh bạ" repo của dự án. Nằm trong repo docs, share cho cả đội qua Git. Khi bạn khai báo thêm repo, script sẽ ghi vào đây và push lên để đồng đội tự động thấy lần sau.

Cấu trúc file

workspace: name: LBaaS created: 2026-06-06 docs_repo: docs handoff_folder: docs/handoff codegraph_mcp: codegraph codegraph_mode: multi-repo repos: # Repo docs — bắt buộc, không build codegraph - name: docs url: https://gitlab.com/fci/lbaas-docs.git branch: main type: docs codegraph: false # App repo — code, build codegraph - name: octavia url: https://gitlab.com/fci/octavia.git branch: main type: app codegraph: true registry_name: octavia # tên dùng khi gọi tool codegraph - name: octavia-dashboard url: https://gitlab.com/fci/octavia-dashboard.git branch: main type: app codegraph: true registry_name: octavia-dashboard

Thêm repo mới vào workspace đang chạy

Khi dự án có thêm repo mới, một thành viên chạy lại script và khai báo repo mới, rồi push workspace.yml. Lần sau đồng đội activate agent sẽ thấy repo mới.

# Chạy lại từ thư mục workspace $ cd ~/work/LBaaS $ ROLE=dev /tmp/fci-kit/init-workspace.sh # Script sẽ hỏi "Khai báo thêm app repo mới?" → y → nhập tên + URL

⚙️ Biến môi trường

Tất cả đều optional. Đặt trước lệnh chạy để bỏ qua bước hỏi tương tác.

BiếnMặc địnhÝ nghĩa
TARGET_DIR.Thư mục workspace root
ROLE(hỏi)po | ba | dev | tester — bỏ bước hỏi role
WORKSPACE_NAMEtên thư mụcTên workspace ghi vào workspace.yml
USER_NAMEusername hệ thốngTên agent dùng khi giao tiếp với bạn
COMM_LANGVietnameseNgôn ngữ giao tiếp của agent
DOC_LANGVietnameseNgôn ngữ viết tài liệu (PRD, story...)
NODE_VERSION22Major Node muốn cài qua nvm (đừng hạ xuống 20)
AUTO_INSTALL_NODE11 = tự cài Node qua nvm nếu thiếu · 0 = chỉ báo lỗi

Ví dụ

# Dev, bỏ qua embed (chỉ build graph, không vector) $ ROLE=dev EMBED_MODEL= TARGET_DIR=~/work/LBaaS \ bash -c "$(curl -fsSL ...)" # Cài lại BMad với custom tên + ngôn ngữ $ USER_NAME="Trọng" COMM_LANG="Vietnamese" DOC_LANG="Vietnamese" \ ROLE=dev ./init-workspace.sh

🔧 Xử lý lỗi thường gặp

❌ NODE_MODULE_VERSION mismatch khi codegraph embed

Error: The module '...better_sqlite3.node' was compiled against a different Node.js version

Nguyên nhân: Node đang dùng không phải Node 22 (codegraph cần prebuilt binary của Node 22).

Giải pháp:

$ nvm install 22 && nvm use 22 # Chạy lại init-workspace.sh

❌ Agent gọi trong Claude Code không thấy codegraph MCP

Nguyên nhân: MCP chưa đăng ký, hoặc đang mở Claude Code ở thư mục khác (không phải workspace root).

Giải pháp:

# Đăng ký lại MCP (lấy đường dẫn tuyệt đối của codegraph) $ CODEGRAPH_BIN=$(which codegraph) $ claude mcp add codegraph -- $CODEGRAPH_BIN mcp --multi-repo # Mở Claude Code đúng thư mục $ cd ~/work/LBaaS && claude

❌ git push bị reject / Authentication failed

Authentication failed: Token hết hạn hoặc sai scope. Tạo lại token trên GitLab và chạy git credential reject để xóa token cũ:

# Xóa credential cũ $ printf "protocol=https\nhost=gitlab.com\n" | git credential reject # Lần push tiếp theo Git sẽ hỏi lại token mới

Push reject (non-fast-forward): Đồng đội đã push trước, repo bị behind. Agent sẽ tự pull --rebase rồi push lại. Nếu vẫn lỗi (conflict thật), Dev resolve thủ công:

$ cd ~/work/LBaaS/docs $ git pull --rebase # Resolve conflict, rồi: $ git rebase --continue && git push

❌ Codegraph kết quả không chính xác sau khi code thay đổi

Nguyên nhân: Graph chưa được rebuild sau khi commit code mới.

# Rebuild graph cho repo cần thiết $ codegraph build octavia $ codegraph build octavia-dashboard

⚠️ Call-graph codegraph không tin cậy trên Python repo (octavia)

Lý do: Python dynamic typing — codegraph không trace được hết edge. Agent được hướng dẫn luôn đối chiếu với grep/ripgrep khi dùng impact trên repo Python. Dùng codegraph để locate (tìm file/symbol), không tin tuyệt đối vào blast radius từ call-graph.

📌 Lưu ý vận hành thường ngày

  • Git không tự thông báo: sau mỗi push/handoff, tự ping Slack hoặc GitLab MR cho người nhận.
  • .codegraph/ nên gitignore: thêm .codegraph/ vào .gitignore của mỗi app repo (chứa graph.db + vectors, không commit).
  • Rebuild codegraph định kỳ: sau sprint merge nhiều code, nên codegraph build <repo> lại để graph chính xác.
